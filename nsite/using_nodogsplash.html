Using NodogsPlash with TollGate

Overview

NodogsPlash was the initial captive portal solution used in TollGate's first demo. It was modified to accept e-cash strings through the username field.

Limitations

  • Required modifications to handle long e-cash strings
  • Had various bugs that led the switch to OpenNDS
  • Limited functionality compared to newer solutions

Implementation Details

  • Used microhttpd as the server
  • Bash backend for processing requests
  • Required custom modifications to core functionality

Legacy Status

While NodogsPlash served as the initial proof of concept, it has been superseded by:

  • OpenNDS for deployments with Forward Authentication Service
  • TollBooth for static frontend implementations

For more information, visit the TollGate project repositories
